Chapter 1: Simplicial Complexes

The combinatorial scaffolding of shape

1.1 Motivation: Why Simplicial Complexes?

The central problem of computational topology is: given a finite set of data points \(P = \{p_1, \ldots, p_n\} \subset \mathbb{R}^d\), infer the topological structure of the underlying space from which the points were sampled. We need a combinatorial model of space that is (a) finite, (b) computable, and (c) rich enough to capture topological information (connectivity, holes, voids).

Simplicial complexes are the answer. They are built from the simplest possible building blocks—points, edges, triangles, tetrahedra, and their higher-dimensional analogues—glued together along shared faces. Every reasonable topological space can be triangulated (represented as a simplicial complex up to homeomorphism), and the topology of a simplicial complex can be computed purely combinatorially, using linear algebra over finite fields.

For our sonification, the simplicial complex is the primary data structure. When a user places or moves points on the canvas, we construct the Vietoris-Rips complex at the current scale parameter. The combinatorial structure of this complex—how many vertices, edges, triangles, and higher simplices it contains, and how they are connected—determines every parameter of the sound.

1.2 The \(k\)-Simplex

Definition 1.1 (Geometric \(k\)-simplex)

Let \(v_0, v_1, \ldots, v_k \in \mathbb{R}^n\) be \(k+1\) points that are affinely independent: the vectors \(v_1 - v_0, v_2 - v_0, \ldots, v_k - v_0\) are linearly independent. The \(k\)-simplex spanned by these points is the convex hull: \[ \sigma = [v_0, v_1, \ldots, v_k] = \left\{ \sum_{i=0}^{k} t_i v_i \;\bigg|\; t_i \geq 0,\; \sum_{i=0}^{k} t_i = 1 \right\}. \] The points \(v_0, \ldots, v_k\) are the vertices of \(\sigma\), and \(k\) is the dimension of \(\sigma\).

The familiar cases:

  • A \(0\)-simplex is a point.
  • A \(1\)-simplex is a line segment (edge) between two points.
  • A \(2\)-simplex is a filled triangle.
  • A \(3\)-simplex is a solid tetrahedron.

The condition of affine independence ensures the simplex is non-degenerate: \(k+1\) points in \(\mathbb{R}^n\) are affinely independent if and only if no point lies in the affine span of the others, which requires \(k \leq n\). The coefficients \(t_i\) are called barycentric coordinates.

Definition 1.2 (Standard \(n\)-simplex)

The standard \(n\)-simplex \(\Delta^n\) is the convex hull of the standard basis vectors \(e_0, e_1, \ldots, e_n \in \mathbb{R}^{n+1}\): \[ \Delta^n = \left\{ (t_0, t_1, \ldots, t_n) \in \mathbb{R}^{n+1} \;\bigg|\; t_i \geq 0,\; \sum_{i=0}^{n} t_i = 1 \right\}. \] Every geometric \(k\)-simplex is affinely isomorphic to \(\Delta^k\).

For computation, we work with abstract simplices: a \(k\)-simplex is simply a set of \(k+1\) vertices, with no geometric embedding. This is the approach taken in our Rust code, where a simplex is represented as a BTreeSet<usize>.

Code Stitch: The Simplex Struct

The implementation in src/topology/simplex.rs represents an abstract simplex as an ordered set of vertex indices. The use of BTreeSet ensures vertices are stored in canonical (sorted) order, which makes equality comparison and hashing well-defined. 

/// A k-simplex: an ordered set of (k+1) vertices.
/// 0-simplex = vertex, 1-simplex = edge, 2-simplex = triangle, etc.
pub struct Simplex {
    // BTreeSet guarantees a canonical ordering of vertices,
    // so that {0,1,2} and {2,0,1} represent the same simplex.
    // This implements the abstract simplex as a finite set.
    pub vertices: BTreeSet<usize>,
}

impl Simplex {
    /// The dimension of the simplex: |vertices| - 1
    /// By Definition 1.1, a k-simplex has k+1 vertices,
    /// so dim(sigma) = |sigma| - 1.
    pub fn dimension(&self) -> isize {
        self.vertices.len() as isize - 1
    }
}

1.3 Faces and the Face Lattice

Definition 1.3 (Face)

A face of a simplex \(\sigma = [v_0, \ldots, v_k]\) is any simplex spanned by a non-empty subset of \(\{v_0, \ldots, v_k\}\). If \(\tau\) is a face of \(\sigma\), we write \(\tau \leq \sigma\). A proper face satisfies \(\tau < \sigma\) (i.e., \(\tau \neq \sigma\)).

A \(k\)-simplex has exactly \(\binom{k+1}{j+1}\) faces of dimension \(j\) for \(0 \leq j \leq k\), giving a total of \(2^{k+1} - 1\) non-empty faces. In particular:

  • An edge \([v_0, v_1]\) has 2 vertex faces and 1 edge face (itself): 3 total.
  • A triangle \([v_0, v_1, v_2]\) has 3 vertices, 3 edges, 1 triangle: 7 total.
  • A tetrahedron \([v_0, v_1, v_2, v_3]\) has 4 vertices, 6 edges, 4 triangles, 1 tetrahedron: 15 total.

Definition 1.4 (Face operator)

The \(i\)-th face operator (or face map) \(d_i\) removes the \(i\)-th vertex: \[ d_i([v_0, \ldots, v_k]) = [v_0, \ldots, \hat{v}_i, \ldots, v_k], \] where \(\hat{v}_i\) denotes omission of \(v_i\). This produces a \((k-1)\)-dimensional face. For a \(k\)-simplex, there are \(k+1\) such codimension-1 faces.

The face operators satisfy the simplicial identity: \(d_i \circ d_j = d_{j+1} \circ d_i\) for \(i \leq j\). This will be crucial when we define the boundary operator in Chapter 2.

Definition 1.5 (Face lattice)

The face lattice (or face poset) of a simplex \(\sigma\) is the partially ordered set \((\mathcal{F}(\sigma), \leq)\) of all faces of \(\sigma\), ordered by inclusion. This is isomorphic to the Boolean lattice \(2^{[k+1]} \setminus \{\emptyset\}\) for a \(k\)-simplex.

Code Stitch: Computing Faces

The faces() method on Simplex computes all codimension-1 faces by removing one vertex at a time. This directly implements Definition 1.4: 

/// Compute all (k-1)-dimensional faces by removing one vertex at a time.
/// Returns pairs of (removed_vertex_index, face) for boundary computation.
///
/// For sigma = [v_0, ..., v_k], we compute
///   d_i(sigma) = [v_0, ..., v_{i-1}, v_{i+1}, ..., v_k]
/// for each i in 0..=k.
///
/// The index i is returned alongside the face because the boundary
/// operator needs it for the sign (-1)^i (Definition 1.4).
pub fn faces(&self) -> Vec<(usize, Simplex)> {
    let verts: Vec<usize> = self.vertices.iter().copied().collect();
    verts
        .iter()
        .enumerate()
        .map(|(i, _)| {
            // Remove the i-th vertex to form the face d_i(sigma)
            let face_verts: BTreeSet<usize> = verts
                .iter()
                .enumerate()
                .filter(|(j, _)| *j != i)
                .map(|(_, &v)| v)
                .collect();
            (i, Simplex { vertices: face_verts })
        })
        .collect()
}

/// Check if this simplex is a face of another.
/// tau <= sigma iff tau.vertices is a subset of sigma.vertices.
/// This implements the face relation of Definition 1.3.
pub fn is_face_of(&self, other: &Simplex) -> bool {
    self.vertices.is_subset(&other.vertices)
}

1.4 Simplicial Complexes

Definition 1.6 (Abstract simplicial complex)

An abstract simplicial complex \(K\) on a vertex set \(V\) is a collection of finite non-empty subsets of \(V\) (called simplices) satisfying two axioms:

  1. Closure under faces: if \(\sigma \in K\) and \(\tau \leq \sigma\) (i.e., \(\tau\) is a face of \(\sigma\)), then \(\tau \in K\).
  2. Non-empty vertices: \(\{v\} \in K\) for every vertex \(v \in V\).

The dimension of \(K\) is \(\dim(K) = \max_{\sigma \in K} \dim(\sigma)\).

The closure axiom is the key requirement: if a triangle \([v_0, v_1, v_2]\) is in \(K\), then all its edges \([v_0,v_1], [v_0,v_2], [v_1,v_2]\) and vertices \([v_0], [v_1], [v_2]\) must also be in \(K\). Intuitively, you cannot have a triangle without its boundary.

Proposition 1.7 (Intersection axiom)

For a geometric simplicial complex (embedded in \(\mathbb{R}^n\)), an additional axiom holds: if \(\sigma, \tau \in K\), then \(\sigma \cap \tau\) is either empty or a common face of both \(\sigma\) and \(\tau\). This ensures simplices "meet nicely."

Example 1.8

Consider the triangle \(K = \{[0], [1], [2], [0,1], [0,2], [1,2], [0,1,2]\}\). This has 3 vertices (0-simplices), 3 edges (1-simplices), and 1 triangle (2-simplex). Removing the 2-simplex \([0,1,2]\) gives the boundary of the triangle: \(\partial K = \{[0], [1], [2], [0,1], [0,2], [1,2]\}\), which is topologically a circle \(S^1\).

Code Stitch: SimplicialComplex

The SimplicialComplex struct in src/topology/complex.rs enforces the closure axiom automatically: inserting any simplex recursively inserts all its faces. 

/// A simplicial complex: a collection of simplices closed under taking faces.
pub struct SimplicialComplex {
    // BTreeSet gives canonical ordering for deterministic iteration.
    simplices: BTreeSet<Simplex>,
    max_dim: isize,
}

impl SimplicialComplex {
    /// Insert a simplex and all its faces (closure property).
    /// This enforces axiom (1) of Definition 1.6: if sigma in K
    /// and tau <= sigma, then tau in K.
    pub fn insert(&mut self, simplex: Simplex) {
        if simplex.dimension() > self.max_dim {
            self.max_dim = simplex.dimension();
        }
        // Recursively insert all codimension-1 faces
        let faces: Vec<Simplex> = simplex.faces()
            .into_iter().map(|(_, f)| f).collect();
        self.simplices.insert(simplex);
        for face in faces {
            if !self.simplices.contains(&face) {
                self.insert(face); // recursive: face's faces too
            }
        }
    }
}

1.5 The \(f\)-Vector and the \(h\)-Vector

Definition 1.9 (\(f\)-vector)

The \(f\)-vector of a \(d\)-dimensional simplicial complex \(K\) is the sequence \(\mathbf{f}(K) = (f_0, f_1, \ldots, f_d)\), where \(f_k\) is the number of \(k\)-simplices in \(K\).

Example 1.10

The filled triangle \([0,1,2]\) has \(\mathbf{f} = (3, 3, 1)\): 3 vertices, 3 edges, 1 triangle.

The hollow tetrahedron (boundary of the tetrahedron, homeomorphic to \(S^2\)) has \(\mathbf{f} = (4, 6, 4)\): 4 vertices, 6 edges, 4 triangles.

The \(f\)-vector is not a topological invariant—different triangulations of the same space may have different \(f\)-vectors. However, the alternating sum of the \(f\)-vector is a topological invariant: the Euler characteristic.

Definition 1.11 (\(h\)-vector)

The \(h\)-vector \(\mathbf{h}(K) = (h_0, h_1, \ldots, h_{d+1})\) is defined by the polynomial identity: \[ \sum_{k=0}^{d+1} h_k \, x^{d+1-k} = \sum_{k=0}^{d+1} f_{k-1} \, (x-1)^{d+1-k}, \] where we set \(f_{-1} = 1\). Equivalently, \[ h_j = \sum_{i=0}^{j} (-1)^{j-i} \binom{d+1-i}{j-i} f_{i-1}. \] The \(h\)-vector encodes the same information as the \(f\)-vector but has nicer algebraic properties.

Theorem 1.12 (Dehn-Sommerville relations)

If \(K\) is a simplicial sphere (i.e., the boundary of a simplicial polytope), then the \(h\)-vector is symmetric: \(h_i = h_{d+1-i}\) for all \(i\).

Remark 1.13 (Sonification relevance)

In our system, the \(f\)-vector is mapped to just intonation ratios. The ratios \(f_1/f_0\), \(f_2/f_1\), etc., determine the harmonic intervals between the oscillator voices in the BettiDrone. For instance, a triangle with \(\mathbf{f} = (3, 3, 1)\) yields the ratio \(3/3 = 1\) (unison) and \(1/3\) (sub-harmonic). The hollow tetrahedron with \(\mathbf{f} = (4, 6, 4)\) yields \(6/4 = 3/2\) (perfect fifth) and \(4/6 = 2/3\) (inverted fifth). Topology generates harmony.

Code Stitch: \(f\)-Vector Computation

/// The number of k-simplices for each dimension. This is the f-vector.
/// f_k = |{sigma in K : dim(sigma) = k}| for k = 0, ..., d.
pub fn f_vector(&self) -> Vec<usize> {
    (0..=self.max_dim)
        .map(|d| self.simplices_of_dim(d).len())
        .collect()
}

1.6 Euler Characteristic

Definition 1.14 (Euler characteristic)

The Euler characteristic of a simplicial complex \(K\) with \(f\)-vector \((f_0, f_1, \ldots, f_d)\) is: \[ \chi(K) = \sum_{k=0}^{d} (-1)^k f_k = f_0 - f_1 + f_2 - f_3 + \cdots \]

Theorem 1.15 (Topological invariance of \(\chi\))

The Euler characteristic is a topological invariant: if \(K\) and \(L\) are simplicial complexes with homeomorphic geometric realizations, then \(\chi(K) = \chi(L)\). That is, \(\chi\) depends only on the topology, not on the particular triangulation.

Proof sketch

This follows from the Euler-Poincaré theorem (Theorem 2.20 in Chapter 2): \(\chi(K) = \sum (-1)^k \beta_k\), where the Betti numbers \(\beta_k\) are topological invariants. The proof of that theorem uses the rank-nullity theorem applied to the boundary operators.

Example 1.16

Some classical Euler characteristics:

  • \(\chi(\text{point}) = 1\)
  • \(\chi(S^1) = 0\) — the circle: 3 vertices, 3 edges gives \(3 - 3 = 0\).
  • \(\chi(S^2) = 2\) — the sphere: the hollow tetrahedron gives \(4 - 6 + 4 = 2\).
  • \(\chi(T^2) = 0\) — the torus.
  • \(\chi(\Sigma_g) = 2 - 2g\) — the orientable surface of genus \(g\).

Code Stitch: Euler Characteristic

/// Euler characteristic: chi = Sum (-1)^k f_k
/// This implements Definition 1.14 directly.
/// The alternating sign is achieved by checking k % 2.
pub fn euler_characteristic(&self) -> isize {
    self.f_vector()
        .iter()
        .enumerate()
        .map(|(k, &fk)| {
            if k % 2 == 0 {
                fk as isize       // even dimensions: +f_k
            } else {
                -(fk as isize)   // odd dimensions: -f_k
            }
        })
        .sum()
}

1.7 Star, Link, and Join

Definition 1.17 (Star)

The star of a simplex \(\sigma\) in \(K\) is the set of all simplices that contain \(\sigma\) as a face: \[ \mathrm{St}(\sigma) = \{ \tau \in K \mid \sigma \leq \tau \}. \] Note: \(\mathrm{St}(\sigma)\) is not itself a simplicial complex (it is not closed under faces), but the closed star \(\overline{\mathrm{St}}(\sigma)\), consisting of \(\mathrm{St}(\sigma)\) together with all faces of simplices in \(\mathrm{St}(\sigma)\), is a subcomplex of \(K\).

Definition 1.18 (Link)

The link of \(\sigma\) in \(K\) is: \[ \mathrm{Lk}(\sigma) = \{ \tau \in \overline{\mathrm{St}}(\sigma) \mid \tau \cap \sigma = \emptyset \}. \] Intuitively, the link is the "boundary of the star" after removing \(\sigma\) itself. The link encodes the local topology around \(\sigma\).

Proposition 1.19

If \(K\) is a triangulation of a manifold \(M\), then for every vertex \(v\), the link \(\mathrm{Lk}(v)\) is a triangulation of \(S^{d-1}\), where \(d = \dim M\). This is the combinatorial analogue of the fact that a manifold is locally Euclidean.

Definition 1.20 (Join)

The join of two simplicial complexes \(K\) and \(L\) on disjoint vertex sets is: \[ K * L = \{ \sigma \cup \tau \mid \sigma \in K,\; \tau \in L \} \cup K \cup L. \] The join of an \(m\)-simplex and an \(n\)-simplex is an \((m+n+1)\)-simplex. For example, the join of two edges (1-simplices) is a tetrahedron (3-simplex).

Code Stitch: Star and Link

/// Star of a simplex sigma: all simplices tau such that sigma is a face of tau.
/// Implements Definition 1.17: St(sigma) = {tau in K | sigma <= tau}.
pub fn star(&self, sigma: &Simplex) -> Vec<&Simplex> {
    self.simplices
        .iter()
        .filter(|tau| sigma.is_face_of(tau))
        .collect()
}

/// Link of a simplex sigma: faces of St(sigma) disjoint from sigma.
/// Implements Definition 1.18: Lk(sigma) = {tau in cl(St(sigma)) | tau cap sigma = empty}.
pub fn link(&self, sigma: &Simplex) -> Vec<&Simplex> {
    let star = self.star(sigma);
    let sigma_verts = &sigma.vertices;
    star.iter()
        .flat_map(|tau| tau.faces().into_iter().map(|(_, f)| f))
        .filter(|face| face.vertices.is_disjoint(sigma_verts))
        .filter(|face| self.simplices.contains(face))
        .collect()
}

1.8 The Vietoris-Rips Complex

Given a finite metric space (or a point cloud \(P \subset \mathbb{R}^d\) with the Euclidean metric), we need a systematic way to build a simplicial complex that captures the topology of the underlying space. The two classical constructions are the Vietoris-Rips complex and the Čech complex.

Definition 1.21 (Vietoris-Rips complex)

Given a finite metric space \((P, d)\) and a scale parameter \(\varepsilon > 0\), the Vietoris-Rips complex \(\mathrm{VR}(P, \varepsilon)\) is the abstract simplicial complex whose \(k\)-simplices are subsets \(\{p_0, \ldots, p_k\} \subseteq P\) with the property that all pairwise distances are at most \(\varepsilon\): \[ \sigma = \{p_0, \ldots, p_k\} \in \mathrm{VR}(P, \varepsilon) \quad \iff \quad d(p_i, p_j) \leq \varepsilon \;\text{ for all } 0 \leq i < j \leq k. \]

The Vietoris-Rips complex is completely determined by its 1-skeleton (the graph of edges): a set of vertices forms a simplex if and only if every pair is connected by an edge. This is the clique complex (or flag complex) of the proximity graph.

Definition 1.22 (Čech complex)

The Čech complex \(\check{C}(P, \varepsilon)\) consists of all subsets \(\{p_0, \ldots, p_k\}\) whose closed \(\varepsilon/2\)-balls have a common intersection: \[ \{p_0, \ldots, p_k\} \in \check{C}(P, \varepsilon) \quad \iff \quad \bigcap_{i=0}^{k} \overline{B}(p_i, \varepsilon/2) \neq \emptyset. \]

Theorem 1.23 (Nerve theorem / Čech vs. Rips interleaving)

By the nerve theorem, the Čech complex has the same homotopy type as the union of balls \(\bigcup_i B(p_i, \varepsilon/2)\). The Vietoris-Rips complex and Čech complex are interleaved: \[ \check{C}(P, \varepsilon) \subseteq \mathrm{VR}(P, \varepsilon) \subseteq \check{C}(P, \sqrt{2}\varepsilon). \] In particular, they agree on the 1-skeleton and on the homotopy type in low dimensions for "generic" point clouds.

Remark 1.24

We use the Vietoris-Rips complex rather than the Čech complex because it is computationally simpler: checking pairwise distances is \(O(n^2)\), while computing common intersections of balls requires geometric predicates. The interleaving theorem guarantees that the topological information we extract is qualitatively correct.

Code Stitch: Vietoris-Rips Construction

The vietoris_rips function builds the complex incrementally: first all vertices, then edges between points within the radius, then triangles (cliques of three edges), then tetrahedra (cliques of six edges). This implements Definition 1.21 up to dimension 3. 

/// Build a Vietoris-Rips complex from a point cloud at a given radius.
/// Implements Definition 1.21: sigma in VR(P, r) iff d(p_i, p_j) <= r
/// for all pairs i, j in sigma.
pub fn vietoris_rips(points: &[[f64; 2]], radius: f64) -> Self {
    let n = points.len();
    let mut complex = Self::new();

    // 0-simplices: every point is a vertex
    for i in 0..n {
        complex.insert_simplex(&[i]);
    }

    // Euclidean distance function
    let dist = |a: &[f64; 2], b: &[f64; 2]| {
        ((a[0] - b[0]).powi(2) + (a[1] - b[1]).powi(2)).sqrt()
    };

    // 1-simplices (edges): insert [i,j] when d(p_i, p_j) <= r
    for i in 0..n {
        for j in (i + 1)..n {
            if dist(&points[i], &points[j]) <= radius {
                complex.insert_simplex(&[i, j]);
            }
        }
    }

    // 2-simplices (triangles): the clique condition —
    // all three edges must exist (all pairwise distances <= r)
    for i in 0..n {
        for j in (i+1)..n {
            if dist(&points[i], &points[j]) > radius { continue; }
            for k in (j+1)..n {
                if dist(&points[i], &points[k]) <= radius
                    && dist(&points[j], &points[k]) <= radius
                {
                    complex.insert_simplex(&[i, j, k]);
                }
            }
        }
    }

    // 3-simplices (tetrahedra): all 6 edges must exist
    // (all C(4,2) = 6 pairwise distances <= r)
    for i in 0..n {
        for j in (i+1)..n {
            for k in (j+1)..n {
                for l in (k+1)..n {
                    let all_close =
                        dist(&points[i], &points[j]) <= radius
                        && dist(&points[i], &points[k]) <= radius
                        && dist(&points[i], &points[l]) <= radius
                        && dist(&points[j], &points[k]) <= radius
                        && dist(&points[j], &points[l]) <= radius
                        && dist(&points[k], &points[l]) <= radius;
                    if all_close {
                        complex.insert_simplex(&[i, j, k, l]);
                    }
                }
            }
        }
    }

    complex
}

1.10 The Boundary Matrix

The boundary matrix is the bridge between the combinatorial structure of the simplicial complex and the algebraic computation of homology (Chapter 2). We introduce it here because it is computed from the complex.

Definition 1.25 (Boundary matrix)

Fix an ordering of the \(k\)-simplices \(\sigma_1^{(k)}, \ldots, \sigma_{f_k}^{(k)}\) and the \((k-1)\)-simplices \(\sigma_1^{(k-1)}, \ldots, \sigma_{f_{k-1}}^{(k-1)}\). The boundary matrix \([\partial_k]\) is the \(f_{k-1} \times f_k\) matrix with entries in the coefficient field (we use \(\mathbb{F}_2 = \mathbb{Z}/2\mathbb{Z}\)) defined by: \[ [\partial_k]_{ij} = \begin{cases} 1 & \text{if } \sigma_i^{(k-1)} \text{ is a face of } \sigma_j^{(k)} \\ 0 & \text{otherwise.} \end{cases} \] Over \(\mathbb{F}_2\), the signs \((-1)^i\) in the oriented boundary formula vanish (since \(-1 \equiv 1 \pmod{2}\)), so we only track incidence.

Code Stitch: Boundary Matrix

/// Compute boundary matrix d_k : C_k -> C_{k-1} over Z/2Z.
/// Returns a sparse representation: list of (row, col) pairs
/// where the matrix entry is 1 mod 2.
///
/// The rows are indexed by (k-1)-simplices, columns by k-simplices.
/// Entry (i, j) = 1 means the i-th (k-1)-simplex is a face of
/// the j-th k-simplex (Definition 1.25).
pub fn boundary_matrix_z2(&self, k: isize) -> BoundaryMatrix {
    let k_simplices = self.simplices_of_dim(k);
    let k_minus_1_simplices = self.simplices_of_dim(k - 1);

    // Build index lookup: (k-1)-simplex -> row index
    let row_index: HashMap<&Simplex, usize> = k_minus_1_simplices
        .iter().enumerate()
        .map(|(i, s)| (*s, i)).collect();

    let rows = k_minus_1_simplices.len();
    let cols = k_simplices.len();
    let mut entries = Vec::new();

    // For each k-simplex (column j), find its faces (row i)
    for (j, sigma) in k_simplices.iter().enumerate() {
        for (_, face) in sigma.faces() {
            if let Some(&i) = row_index.get(&face) {
                entries.push((i, j)); // M[i][j] = 1 mod 2
            }
        }
    }

    BoundaryMatrix { rows, cols, entries }
}

Why Each Concept Matters: Applications Trace

Every definition and theorem in this chapter exists for a reason. Here we trace each concept to its applications — in our sonification engine, in the broader mathematical landscape, and in the sciences. The goal is to make visible the thread from abstract definition to concrete utility.

Application Trace: The Simplex (Definitions 1.1–1.2)

Why simplices? A \(k\)-simplex is the simplest possible \(k\)-dimensional geometric object: a point, an edge, a triangle, a tetrahedron. Any convex polytope can be decomposed into simplices (triangulation), and any topological space can be approximated by a simplicial complex (simplicial approximation theorem). Simplices are the “atoms” of discrete geometry.

In engineering: Finite element methods (FEM) decompose domains into simplices (triangular meshes in 2D, tetrahedral meshes in 3D) to solve PDEs numerically. Computer graphics uses triangle meshes universally. Delaunay triangulations are simplicial complexes optimized for interpolation quality.

In our engine: The Simplex struct is the atomic building block. Its dimension() method (Definition 1.1: dimension = |vertices| − 1) determines how high-dimensional the topology can get. Its faces() method (Definition 1.4) is the workhorse for boundary computation, which in turn drives all of homology.

Application Trace: Faces and the Face Lattice (Definitions 1.3–1.5)

Why faces? The face relation \(\tau \leq \sigma\) (Definition 1.3) organizes simplices into a partial order. The face lattice (Definition 1.5) is this poset — it encodes the complete combinatorial structure of a simplicial complex. Two complexes are isomorphic as abstract simplicial complexes if and only if their face lattices are isomorphic as posets.

Why the face operator \(d_i\)? The \(i\)-th face operator \(d_i([v_0, \ldots, v_k]) = [v_0, \ldots, \hat{v}_i, \ldots, v_k]\) (Definition 1.4) removes the \(i\)-th vertex. The alternating sum \(\partial_k = \sum (-1)^i d_i\) becomes the boundary operator in Chapter 2. The signs \((-1)^i\) are not arbitrary: they encode the orientation of each face relative to the simplex. Without the correct signs, \(\partial^2 \neq 0\) and homology would be undefined.

In our engine: Simplex::faces() returns pairs (i, face) where \(i\) is the index of the removed vertex. The index \(i\) is propagated to the boundary matrix construction in boundary_matrix_z2(), where it determines the sign (though over \(\mathbb{F}_2\), all signs are 1).

Application Trace: The Simplicial Complex (Definition 1.6)

Why the closure axiom? Definition 1.6 requires that if \(\sigma \in K\) and \(\tau \leq \sigma\), then \(\tau \in K\). This is not a technical convenience — it is a topological necessity. Without it, the geometric realization \(|K|\) would have “dangling” simplices: a triangle whose edges aren’t in the complex, which is geometrically nonsensical. The closure axiom ensures that \(|K|\) is a well-defined topological space.

In computer science: Simplicial complexes model: hypergraphs (higher-order networks beyond pairwise interactions), database schemas (the nerve of a relational schema is a simplicial complex), and independence systems in combinatorial optimization.

In our engine: SimplicialComplex::insert() automatically inserts all faces of any added simplex, enforcing the closure axiom. This is the function that makes the complex well-formed. Without it, the boundary matrices would have missing rows and the homology computation would be incorrect.

Application Trace: The \(f\)-vector and \(h\)-vector (Definitions 1.9, 1.11)

Why the \(f\)-vector? The \(f\)-vector \((f_0, f_1, \ldots, f_d)\) counts simplices by dimension: \(f_0\) vertices, \(f_1\) edges, \(f_2\) triangles, etc. It is the crudest “fingerprint” of a complex — not a topological invariant (different triangulations of the same space have different \(f\)-vectors), but a combinatorial invariant that constrains the topology.

Why the \(h\)-vector? The \(h\)-vector is a linear transformation of the \(f\)-vector that has better algebraic properties. The Dehn-Sommerville relations (Theorem 1.12) constrain the \(h\)-vector of a simplicial sphere, which constrains which \(f\)-vectors are realizable. This is the starting point of the \(g\)-conjecture (now theorem, proved by Adiprasito 2018) — one of the deepest results in combinatorial topology.

In our engine: SimplicialComplex::f_vector() drives the just-intonation tuning of the BettiDrone. The ratios \(f_1/f_0\), \(f_2/f_1\), etc. become frequency ratios between oscillators. This mapping is natural: the \(f\)-vector measures the “combinatorial density” at each dimension, and frequency ratios are the harmonic-series analogue of density. A complex with many edges relative to vertices (\(f_1/f_0\) large) produces a wider harmonic spread.

Application Trace: Euler Characteristic (Definition 1.14, Theorem 1.15)

Why is \(\chi\) topologically invariant? The Euler characteristic \(\chi = \sum (-1)^k f_k\) depends on the specific triangulation, yet Theorem 1.15 says it doesn’t change under refinement. This is remarkable: you can subdivide a triangle into thousands of sub-triangles, change every \(f_k\), and \(\chi\) remains the same. The proof (deferred to Chapter 2) uses the Euler-Poincaré theorem: \(\chi = \sum (-1)^k \beta_k\), and the Betti numbers are manifestly topological invariants.

Applications across mathematics:

  • Gauss-Bonnet: \(\int K\,dA = 2\pi\chi\). Total Gaussian curvature is topological.
  • Index theory: The Atiyah-Singer index theorem generalizes \(\chi\) to elliptic operators on manifolds, connecting analysis, geometry, and topology.
  • Euler calculus: Schapira, Viro, and Ghrist developed integration with respect to Euler characteristic (\(\int f\,d\chi\)), with applications to sensor networks and target counting.
  • Algebraic geometry: \(\chi\) of a variety controls its arithmetic via the Weil conjectures (proved by Deligne 1974).

In our engine: \(\chi\) is the single scalar that parameterizes CurvatureReverb. Via Gauss-Bonnet, positive \(\chi\) means positive average curvature (converging geodesics, focused sound), negative \(\chi\) means negative curvature (diverging geodesics, diffuse sound). This is the most physically motivated mapping in the entire engine.

Application Trace: Star and Link (Definitions 1.17–1.18)

Why star and link? The star \(\mathrm{St}(\sigma)\) collects everything “touching” \(\sigma\); the link \(\mathrm{Lk}(\sigma)\) is the boundary of this neighborhood. The link is the local “shape” of the complex near \(\sigma\).

The link detects manifold structure: A simplicial complex is a combinatorial manifold if and only if the link of every vertex is a sphere (or a disk at the boundary). If \(\mathrm{Lk}(v)\) is a circle, the complex looks like a surface near \(v\). If \(\mathrm{Lk}(v)\) is a sphere \(S^2\), it looks like a 3-manifold near \(v\). Non-manifold points (singularities) have exotic links.

In data analysis: The link of a vertex in a VR complex tells you about the local topology of the data near that point. This is used in local homology estimation, manifold learning (detecting the local dimension), and singularity detection.

In our engine: star() and link() are available for future extensions: per-vertex curvature estimation (via the angle defect formula \(\kappa(v) = 2\pi - \sum_{\sigma \ni v} \theta_\sigma(v)\)) and local sonification parameters that vary spatially across the complex.

Application Trace: Vietoris-Rips Complex (Definitions 1.21–1.22, Theorem 1.23)

Why VR and not Čech? Both constructions build a simplicial complex from a point cloud at a scale \(r\). The Čech complex uses the nerve of balls \(B(p_i, r)\) and is theoretically ideal (the Nerve theorem guarantees homotopy equivalence to the union of balls). But Čech is computationally expensive: determining whether \(k+1\) balls have a common intersection point requires solving a minimax problem in \(\mathbb{R}^d\).

The VR complex only checks pairwise distances: \(\sigma \in \mathrm{VR}(P, r)\) iff all pairwise distances within \(\sigma\) are \(\leq r\). This is a flag complex (determined entirely by its 1-skeleton), so it can be built from the pairwise distance matrix alone. The price: VR is a coarser approximation. The interleaving theorem (1.23) quantifies the error: \(\check{C}(P, r) \subseteq \mathrm{VR}(P, r) \subseteq \check{C}(P, 2r)\).

In practice: VR is the universal default in TDA software (Ripser, GUDHI, Dionysus) because the flag complex property enables dramatic algorithmic optimizations. Our vietoris_rips() implements the naive construction (check all \(k\)-tuples); Ripser achieves the same result orders of magnitude faster using implicit matrix representation and apparent pairs.

In our engine: SimplicialComplex::vietoris_rips() is the entry point for point cloud sonification. The user clicks points on the canvas, adjusts the radius slider, and the VR complex updates in real time. The radius parameter is the filtration parameter of Chapter 3 — sweeping it produces the persistence diagram.

Application Trace: The Boundary Matrix (Definition 1.25)

Why the boundary matrix? The boundary matrix \([\partial_k]\) is the computational representation of the boundary operator \(\partial_k: C_k \to C_{k-1}\). Once we have the boundary matrices for all dimensions, all of homology reduces to linear algebra: Betti numbers come from ranks, persistence pairs come from column reduction, and the Hodge Laplacian is \(L_k = \partial_{k+1}\partial_{k+1}^T + \partial_k^T\partial_k\).

The matrix encodes topology as algebra: Every topological question about the complex can be answered by matrix operations on \([\partial_k]\). This is the fundamental insight of computational topology: topology is computable because it reduces to linear algebra.

In our engine: boundary_matrix_z2() produces the sparse matrix whose rank, computed by rank_z2(), feeds into the Betti number calculation. The entire downstream audio engine — every instrument, every effect — ultimately depends on the correctness of this matrix.

Exercises

Exercise 1.1

Compute the \(f\)-vector and Euler characteristic of the octahedron (the boundary of the regular octahedron in \(\mathbb{R}^3\), which triangulates \(S^2\)).

Exercise 1.2

Show that for any \(d\)-dimensional simplicial complex \(K\), the number of edges satisfies \(f_1 \leq \binom{f_0}{2}\). When does equality hold? What does this mean for the Vietoris-Rips complex?

Exercise 1.3

Let \(P = \{(0,0), (1,0), (0,1), (1,1)\}\) be the vertices of the unit square. For what values of the radius \(\varepsilon\) does \(\mathrm{VR}(P, \varepsilon)\) have: (a) 4 components, (b) 1 component with a loop, (c) 1 component with no loop? Compute \(\mathbf{f}\) and \(\chi\) at each stage.

Exercise 1.4

Prove that the Vietoris-Rips complex is a flag complex: \(\sigma \in \mathrm{VR}(P,\varepsilon)\) if and only if every edge of \(\sigma\) is in \(\mathrm{VR}(P,\varepsilon)\). Show that the Čech complex does not have this property in general.

Exercise 1.5

The Kruskal count of a simplicial complex is \(\sum_{k=0}^{d} f_k\). Compute this for the Vietoris-Rips complex of \(n\) evenly-spaced points on a circle of radius 1, as a function of \(\varepsilon\). At what \(\varepsilon\) is the count maximized?

Exercise 1.6 (Sonification)

The BettiDrone uses the ratios \(f_{k+1}/f_k\) from the \(f\)-vector as just intonation intervals. Compute these ratios for: (a) the filled triangle, (b) the hollow tetrahedron, (c) the octahedron. Which produces the most "consonant" harmony, and why?

Exercise 1.7 (Implementation)

The current vietoris_rips implementation has time complexity \(O(n^4)\) (it checks all 4-tuples for tetrahedra). Describe an algorithm using the flag complex property (Exercise 1.4) that achieves \(O(n^2 + |\text{VR}|)\) time, where \(|\text{VR}|\) is the number of simplices in the output. Hint: use the Bron-Kerbosch algorithm for maximal cliques.

The material in this chapter is standard and can be found in any introduction to algebraic topology or combinatorial topology. The following are the canonical references, ordered roughly by accessibility.

Primary Textbooks

  1. Hatcher, A. Algebraic Topology. Cambridge University Press, 2002. Chapter 2 (Homology) begins with an excellent treatment of simplicial and singular complexes. Freely available at pi.math.cornell.edu/~hatcher/AT/ATpage.html.
    The standard graduate text. Our definitions of simplicial complex, face, and star follow Hatcher's conventions.
  2. Munkres, J.R. Elements of Algebraic Topology. Addison-Wesley, 1984. ISBN 0-201-04586-9.
    Chapters 1–5 give a thorough, self-contained treatment of simplicial complexes and simplicial homology. More elementary than Hatcher; carefully develops the theory with explicit computations. Our treatment of the f-vector and Euler characteristic follows Munkres.
  3. Spanier, E.H. Algebraic Topology. Springer, 1966. ISBN 978-0-387-94426-5.
    The encyclopaedic reference. Chapter 3 treats simplicial complexes in full generality. Consult for the relationship between abstract and geometric simplicial complexes.
  4. Rotman, J.J. An Introduction to Algebraic Topology. Springer GTM 119, 1988.
    Chapters 7–8 develop simplicial complexes and homology with a focus on computation.

  1. Edelsbrunner, H. and Harer, J.L. Computational Topology: An Introduction. AMS, 2010. ISBN 978-0-8218-4925-5.
    Part I (chapters 1–4) covers simplicial complexes from the computational perspective. The Vietoris-Rips and Čech constructions are in Chapter III. Our vietoris_rips() implementation follows their definition.
  2. Zomorodian, A. Topology for Computing. Cambridge University Press, 2005. ISBN 0-521-83666-2.
    Chapters 4–6 cover simplicial complexes as computational objects. Explicit algorithms for boundary matrix computation.
