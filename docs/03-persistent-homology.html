Chapter 3: Persistent Homology and Filtrations

Multi-scale topology and the persistence diagram

3.1 The Scale Problem

Chapter 1 showed how to build a Vietoris-Rips complex \(\mathrm{VR}(P, \varepsilon)\) from a point cloud at a fixed radius \(\varepsilon\). But which \(\varepsilon\) should we choose?

  • If \(\varepsilon\) is too small, every point is isolated: \(\beta_0 = n\), \(\beta_k = 0\) for \(k \geq 1\).
  • If \(\varepsilon\) is too large, everything is connected and filled: \(\beta_0 = 1\), \(\beta_k = 0\) for \(k \geq 1\).
  • At intermediate scales, the complex reveals the "true" topology of the underlying space.

Persistent homology solves this problem by considering all scales simultaneously. Instead of picking a single \(\varepsilon\), we sweep \(\varepsilon\) from 0 to \(\infty\) and track how topological features (components, loops, voids) appear and disappear. Features that persist across a wide range of scales are "real" (signal); features that appear and disappear quickly are "noise."

For sonification, persistence provides a temporal dimension: the filtration parameter becomes time, and the births and deaths of features become musical events. The persistence diagram is literally a score.

3.2 Filtrations

Definition 3.1 (Filtration)

A filtration of a simplicial complex \(K\) is a nested sequence of subcomplexes: \[ \emptyset = K_0 \subseteq K_1 \subseteq K_2 \subseteq \cdots \subseteq K_n = K, \] where each \(K_i\) is a simplicial complex and the inclusions are simplicial maps. Equivalently, a filtration assigns to each simplex \(\sigma \in K\) a filtration value \(f(\sigma) \in \mathbb{R}\) such that \(f(\tau) \leq f(\sigma)\) whenever \(\tau \leq \sigma\) (faces enter no later than their cofaces).

Definition 3.2 (Vietoris-Rips filtration)

The Vietoris-Rips filtration of a point cloud \(P\) is the filtration where \(K_\varepsilon = \mathrm{VR}(P, \varepsilon)\). Each simplex \(\sigma = \{p_0, \ldots, p_k\}\) enters the filtration at the value: \[ f(\sigma) = \max_{0 \leq i < j \leq k} d(p_i, p_j), \] the diameter of the simplex. In particular:

  • Each vertex enters at \(f = 0\).
  • An edge \([p_i, p_j]\) enters at \(f = d(p_i, p_j)\).
  • A triangle \([p_i, p_j, p_k]\) enters at \(f = \max(d(p_i,p_j), d(p_i,p_k), d(p_j,p_k))\).

3.3 Persistent Homology

As we increase \(\varepsilon\) through the filtration, the homology groups \(H_k(K_\varepsilon)\) change. New features are born (a new component appears, a loop forms, a void closes) and old features die (components merge, loops are filled in, voids are capped off).

Definition 3.3 (Persistent homology group)

For \(i \leq j\), the \((i,j)\)-persistent \(k\)-th homology group is: \[ H_k^{i,j} = Z_k(K_i) / (B_k(K_j) \cap Z_k(K_i)), \] which captures the \(k\)-cycles born at or before step \(i\) that are still alive (not yet boundaries) at step \(j\). The inclusion \(K_i \hookrightarrow K_j\) induces a map \(H_k(K_i) \to H_k(K_j)\), and \(H_k^{i,j}\) is its image.

Definition 3.4 (Birth and death)

A homology class \(\gamma \in H_k(K_i)\) is born at step \(i\) if \(\gamma \notin H_k^{i-1,i}\)—it is not in the image of the map from \(K_{i-1}\). The class dies at step \(j\) if the image of \(\gamma\) becomes zero in \(H_k^{i,j}\) but was nonzero in \(H_k^{i,j-1}\).

Definition 3.5 (Persistence pair)

A persistence pair is a pair \((b, d)\) where \(b\) is the birth time and \(d\) is the death time of a homology class. The persistence of the pair is: \[ \mathrm{pers}(b, d) = d - b. \] Long persistence indicates a "significant" topological feature; short persistence indicates noise.

Definition 3.6 (Essential class)

If a class is born but never dies (it persists to \(\varepsilon = \infty\)), it is called an essential class. We write its death as \(d = \infty\) and its persistence as \(\infty\). Essential classes correspond to the "true" topology of the underlying space.

Code Stitch: PersistencePair

/// A persistence pair (birth, death) representing a topological feature
/// that appears at `birth` and disappears at `death`.
/// Implements Definitions 3.5 and 3.6.
pub struct PersistencePair {
    pub birth: f64,       // filtration value at birth (b)
    pub death: f64,       // filtration value at death (d), or f64::INFINITY
    pub dimension: usize, // homological dimension k
}

impl PersistencePair {
    /// The lifetime of the feature: pers(b,d) = d - b.
    pub fn persistence(&self) -> f64 {
        if self.death.is_infinite() { f64::INFINITY }
        else { self.death - self.birth }
    }

    /// Is this an essential class (Definition 3.6)?
    pub fn is_essential(&self) -> bool {
        self.death.is_infinite()
    }
}

3.4 The Persistence Diagram

Definition 3.7 (Persistence diagram)

The persistence diagram \(\mathrm{Dgm}(K)\) is the multiset of all persistence pairs \(\{(b_i, d_i)\}\) in the upper half-plane \(\{(b,d) \mid b \leq d\}\). Points near the diagonal \(d = b\) represent short-lived (noisy) features; points far from the diagonal represent persistent (significant) features. Essential classes are represented as points at \((b, \infty)\).

Blue dots are \(H_1\) pairs (loops forming and filling). Star = essential class.

3.5 The Stability Theorem

Definition 3.8 (Bottleneck distance)

The bottleneck distance between two persistence diagrams \(\mathrm{Dgm}_1\) and \(\mathrm{Dgm}_2\) is: \[ d_B(\mathrm{Dgm}_1, \mathrm{Dgm}_2) = \inf_\gamma \sup_{x \in \mathrm{Dgm}_1} \|x - \gamma(x)\|_\infty, \] where \(\gamma\) ranges over all bijections between the two diagrams (with points on the diagonal included to allow unmatched points).

Definition 3.9 (Wasserstein distance)

The \(p\)-Wasserstein distance between persistence diagrams is: \[ W_p(\mathrm{Dgm}_1, \mathrm{Dgm}_2) = \left(\inf_\gamma \sum_{x \in \mathrm{Dgm}_1} \|x - \gamma(x)\|_\infty^p\right)^{1/p}. \] The bottleneck distance is the limit \(W_\infty\).

Theorem 3.10 (Stability theorem, Cohen-Steiner, Edelsbrunner, Harer 2007)

Let \(f, g : K \to \mathbb{R}\) be two filtration functions on the same simplicial complex \(K\). Then: \[ d_B(\mathrm{Dgm}(f), \mathrm{Dgm}(g)) \leq \|f - g\|_\infty. \] In words: the persistence diagram is 1-Lipschitz with respect to the bottleneck distance. Small perturbations to the input produce small perturbations to the output.

Proof sketch

The key idea is the box lemma: if \(\|f - g\|_\infty \leq \delta\), then for every persistence pair \((b, d)\) in \(\mathrm{Dgm}(f)\), there exists a pair \((b', d')\) in \(\mathrm{Dgm}(g)\) with \(|b - b'| \leq \delta\) and \(|d - d'| \leq \delta\), or the pair has persistence \(d - b \leq 2\delta\) (and can be matched to the diagonal). This is proved by tracking how the inclusion maps between sublevel sets change under perturbation of the filtration function. The detailed proof uses the algebraic stability of the persistence module structure. \(\square\)

Remark 3.11 (Importance for sonification)

The stability theorem guarantees that our sonification is continuous: when the user moves a point slightly, the persistence diagram changes slightly, and therefore the sound changes smoothly. Without stability, small mouse movements could cause dramatic jumps in the audio—a poor user experience. Stability is what makes persistent homology suitable for real-time interactive sonification.

3.6 Persistence Landscapes

Definition 3.12 (Persistence landscape)

Given a persistence diagram \(\mathrm{Dgm}\) in dimension \(k\), the \(j\)-th persistence landscape function \(\lambda_j : \mathbb{R} \to \mathbb{R}\) is defined as follows. For each pair \((b_i, d_i)\), define the "tent function": \[ \Lambda_i(t) = \max\left(0, \min\left(t - b_i, \; d_i - t\right)\right). \] This is a triangle of height \((d_i - b_i)/2\) centered at the midpoint \((b_i + d_i)/2\). Then \(\lambda_j(t)\) is the \(j\)-th largest value among all \(\Lambda_i(t)\) at parameter \(t\).

Landscapes convert the multiset of persistence pairs into a sequence of functions, which are easier to average, compute norms of, and use in statistical tests. They are Lipschitz-stable (like persistence diagrams) and embed naturally into Banach spaces.

Code Stitch: Landscape Computation

/// Persistence landscape value at a given point.
/// lambda_k(t) for the k-th landscape function at parameter t.
///
/// For each pair (b_i, d_i), the tent function Lambda_i(t) is:
///   Lambda_i(t) = max(0, min(t - b_i, d_i - t))
/// Then lambda_k(t) = k-th largest Lambda_i(t).
pub fn landscape(&self, dim: usize, k: usize, t: f64) -> f64 {
    let mut values: Vec<f64> = self
        .pairs_in_dim(dim)
        .iter()
        .filter(|p| !p.is_essential())
        .map(|p| {
            let mid = p.midpoint();
            let half_life = p.persistence() / 2.0;
            // Tent function: Lambda_i(t)
            if (t - mid).abs() < half_life {
                half_life - (t - mid).abs()
            } else {
                0.0
            }
        })
        .collect();
    // Sort descending to get the k-th largest
    values.sort_by(|a, b| b.partial_cmp(a).unwrap());
    values.get(k).copied().unwrap_or(0.0)
}

3.7 Persistence Entropy

Definition 3.13 (Persistence entropy)

The persistence entropy of a persistence diagram is the Shannon entropy of the normalized persistence values: \[ H = -\sum_{i} p_i \log p_i, \quad \text{where} \quad p_i = \frac{\mathrm{pers}(b_i, d_i)}{\sum_j \mathrm{pers}(b_j, d_j)}. \] The sum ranges over all finite persistence pairs. High entropy indicates many features of similar persistence (uniform distribution); low entropy indicates a few dominant features.

Remark 3.14

Persistence entropy provides a single scalar summary of the complexity of the persistence diagram. For sonification, it can modulate global parameters like overall brightness, density, or chaos of the sound.

Code Stitch: Persistence Entropy

/// Persistence entropy: H = -Sum p_i log(p_i)
/// where p_i = pers_i / total_pers (Definition 3.13).
pub fn persistence_entropy(&self) -> f64 {
    let total = self.total_persistence();
    if total == 0.0 { return 0.0; }
    self.pairs.iter()
        .filter(|p| !p.is_essential())
        .map(|p| {
            let prob = p.persistence() / total;
            if prob > 0.0 { -prob * prob.ln() }
            else { 0.0 }
        })
        .sum()
}

3.8 The Column Reduction Algorithm

The standard algorithm for computing persistence is a variant of Gaussian elimination called column reduction. It processes the boundary matrix of the entire filtered complex in a specific order (by filtration value) and produces the persistence pairs directly.

Algorithm 3.15 (Standard persistence algorithm)

Input: A filtered simplicial complex \(K\) with simplices ordered by filtration value: \(\sigma_1, \sigma_2, \ldots, \sigma_m\).

Output: Persistence pairs.

  1. Build the boundary matrix \(D\) where column \(j\) represents \(\partial(\sigma_j)\) in terms of previously-entered simplices.
  2. For \(j = 1, \ldots, m\):
    • While there exists \(j' < j\) with \(\mathrm{low}(j') = \mathrm{low}(j)\): add column \(j'\) to column \(j\) (over \(\mathbb{F}_2\)).
  3. After reduction:
    • If column \(j\) is nonzero with \(\mathrm{low}(j) = i\), then \((\sigma_i, \sigma_j)\) form a persistence pair: \(\sigma_i\) is born and \(\sigma_j\) kills it. Record \((f(\sigma_i), f(\sigma_j))\).
    • If column \(j\) is zero and row \(j\) is not the \(\mathrm{low}\) of any column, then \(\sigma_j\) creates an essential class.

Remark 3.16 (Our implementation)

Our implementation in persistence.rs takes a simpler approach: it samples the filtration at discrete steps, computes Betti numbers at each step, and tracks births and deaths by comparing consecutive Betti numbers. This is less efficient than the standard algorithm but conceptually transparent and sufficient for small to moderate point clouds (up to ~50 points) in a real-time interactive setting.

3.9 Computing Persistence from Point Clouds

Code Stitch: PersistenceDiagram::from_point_cloud

/// Compute a persistence diagram from a Vietoris-Rips filtration.
/// We sample the filtration at discrete radii and track births/deaths.
///
/// Algorithm:
///   1. Compute max pairwise distance to determine filtration range.
///   2. For each step eps = k * (max_dist / num_steps):
///      a. Build VR(P, eps).
///      b. Compute Betti numbers.
///      c. Compare with previous step:
///         - If beta_k increased: new features born at eps.
///         - If beta_k decreased: features died at eps.
///   3. Remaining active features at the end are essential.
pub fn from_point_cloud(points: &[[f64; 2]], num_steps: usize) -> Self {
    // Find filtration range
    let mut max_dist: f64 = 0.0;
    for i in 0..points.len() {
        for j in (i+1)..points.len() {
            let d = ((points[i][0]-points[j][0]).powi(2)
                   + (points[i][1]-points[j][1]).powi(2)).sqrt();
            if d > max_dist { max_dist = d; }
        }
    }

    let step_size = max_dist / num_steps as f64;
    let mut prev_betti = BettiNumbers { values: vec![] };
    let mut active_features: Vec<Vec<(f64, usize)>> = Vec::new();
    let mut pairs = Vec::new();

    for step in 0..=num_steps {
        let radius = step as f64 * step_size;
        let complex = SimplicialComplex::vietoris_rips(points, radius);
        let betti = BettiNumbers::compute(&complex);

        // Compare with previous Betti numbers
        for dim in 0..betti.values.len() {
            let prev = prev_betti.values.get(dim).copied().unwrap_or(0);
            let curr = betti.values[dim];

            if curr > prev {
                // Birth: new features at this radius
                for _ in 0..(curr - prev) {
                    active_features[dim].push((radius, 1));
                }
            } else if curr < prev {
                // Death: features killed at this radius
                for _ in 0..(prev - curr) {
                    if let Some((birth, _)) = active_features[dim].pop() {
                        pairs.push(PersistencePair {
                            birth, death: radius, dimension: dim,
                        });
                    }
                }
            }
        }
        prev_betti = betti;
    }

    // Remaining features are essential (Definition 3.6)
    for (dim, features) in active_features.iter().enumerate() {
        for &(birth, _) in features {
            pairs.push(PersistencePair {
                birth, death: f64::INFINITY, dimension: dim,
            });
        }
    }

    // Sort by persistence (longest-lived first)
    pairs.sort_by(|a, b| b.persistence().partial_cmp(&a.persistence()).unwrap());

    Self { pairs }
}

3.10 Worked Example: Circle of Points

Example 3.17

Consider 8 points evenly spaced on the unit circle: \(p_k = (\cos(2\pi k/8), \sin(2\pi k/8))\) for \(k = 0, \ldots, 7\). The nearest-neighbor distance is \(2\sin(\pi/8) \approx 0.765\).

Filtration stages:

  1. \(\varepsilon = 0\): 8 isolated points. \(\beta_0 = 8\), \(\beta_1 = 0\).
  2. \(\varepsilon = 0.765\): nearest-neighbor edges appear. A cycle of 8 edges forms. \(\beta_0 = 1\) (one component), \(\beta_1 = 1\) (one loop!).
  3. \(\varepsilon = 1.414\): next-nearest-neighbor edges (\(d = 2\sin(\pi/4) = \sqrt{2}\)). Triangles begin to fill the interior. \(\beta_1\) starts to decrease as the loop is partially filled.
  4. \(\varepsilon = 2.0\): the complex is nearly complete. All triangles are filled. \(\beta_0 = 1\), \(\beta_1 = 0\).

Persistence diagram:

  • \(H_0\): Seven pairs with small persistence (components merging) and one essential class.
  • \(H_1\): One pair with high persistence \(\approx (0.765, 1.414)\), representing the "real" loop (the hole in the middle of the circle). This is the dominant feature.

For sonification: the persistent \(H_1\) feature creates a sustained LFO modulation. The short-lived \(H_0\) features create brief transient clicks as components merge. The birth/death events drive the MorseSynth score.

3.11 From Persistence to Sound

The persistence diagram provides two primary sonification channels:

1. Persistence as Delay

Each persistence pair \((b, d)\) maps to a tap in the PersistenceDelay effect:

  • The delay time is proportional to the birth value \(b\): features that appear early have short delays; features that appear late have long delays.
  • The feedback amount is proportional to the persistence \(d - b\): long-lived features echo longer.
  • The level is inversely proportional to the dimension: \(H_0\) features are loudest, \(H_1\) features softer, \(H_2\) softest.

2. Persistence as Score (Morse Theory)

The filtration is interpreted as a temporal axis. As we sweep \(\varepsilon\):

  • Birth events trigger note-on: a new oscillator starts at a frequency derived from the filtration value (log-mapped to a pitch range).
  • Death events trigger note-off: the oscillator enters its release phase, with release time proportional to the feature's persistence.
  • The dimension determines the waveform: \(H_0 \to\) sine, \(H_1 \to\) triangle, \(H_2 \to\) sawtooth.

This is the connection to Morse theory: the filtration function plays the role of a Morse function \(f : M \to \mathbb{R}\), and the critical points (births and deaths) are the "notes" of the score. The Morse inequalities \(c_k \geq \beta_k\) guarantee that the number of critical points is at least the number of topological features.

Why Each Result Matters: Applications Trace

Application Trace: Persistence Pairs and Birth-Death (Definitions 3.4–3.5)

What is a persistence pair? A pair \((b, d)\) records a topological feature (a homology class) that is born at filtration value \(b\) and dies at value \(d\). The feature exists in all complexes \(K_t\) for \(b \leq t < d\). The persistence \(d - b\) measures the feature's lifetime — how wide a range of scales it survives.

Why persistence is a proxy for significance: The fundamental heuristic of TDA is that long-lived features are “signal” and short-lived features are “noise.” A loop that appears at \(r = 0.3\) and dies at \(r = 2.1\) (persistence 1.8) is likely a genuine topological feature of the underlying space. A loop that appears at \(r = 1.0\) and dies at \(r = 1.05\) (persistence 0.05) is likely an artifact of the sampling.

In our engine: Each persistence pair maps to a delay tap in PersistenceDelay. Longer persistence → higher feedback (louder, longer echo). Short-lived pairs produce faint, quickly-decaying echoes. The persistence literally determines how long the feature “rings.”

Application Trace: Essential Classes (Definition 3.6)

What are essential classes? An essential class is a homology class that is born at some filtration value and never dies — it persists to \(d = \infty\). These correspond to the “true” topology of the underlying space (or of the densest complex in the filtration).

Why they matter: For a VR filtration on \(n\) points, there is always exactly one essential \(H_0\) class (the single connected component at the end). Any additional essential classes in higher dimensions indicate genuine topological features of the data that survive at all scales — the most robust features.

In our engine: Essential classes in MorseSynth produce voices that never receive a “note off” event: sustained drones that persist throughout the playback. They represent the permanent topological substrate of the space, sonified as a continuous harmonic foundation.

Application Trace: Bottleneck and Wasserstein Distances (Definitions 3.8–3.9)

What are these distances? The bottleneck distance \(d_B\) between two persistence diagrams is the infimum over all matchings of the maximum cost of any matched pair. The Wasserstein distance \(W_p\) is the \(p\)-th root of the infimum over all matchings of the sum of \(p\)-th powers of costs. Both measure how “different” two persistence diagrams are.

Why they matter: These distances turn the space of persistence diagrams into a metric space, enabling: (1) comparison — how topologically similar are two datasets? (2) averaging — what is the “typical” diagram of a collection? (3) machine learning — kernel methods and feature maps on diagrams. The stability theorem (Theorem 3.10) uses the bottleneck distance.

In our engine: The Wasserstein distance between consecutive frames of a time-varying point cloud could drive a “rate of topological change” parameter: rapid topological change → fast modulation, timbral roughness; slow change → smooth evolution. This is specified as Exercise 3.7 (sonification design).

Computational note: Computing \(d_B\) reduces to a matching problem on a bipartite graph, solvable in \(O(n^{1.5} \log n)\) via the Hopcroft-Karp algorithm. Computing \(W_p\) reduces to optimal transport, solvable via the Hungarian algorithm in \(O(n^3)\) or via Sinkhorn iterations for approximate solutions.

Application Trace: Filtrations and the Scale Problem

The deepest conceptual contribution of persistent homology is solving the scale selection problem. In classical topology, you must choose a single threshold \(r\) to build the Vietoris-Rips complex \(\mathrm{VR}(X, r)\). Too small: everything is disconnected. Too large: everything collapses to a single simplex. There is no canonical choice.

Persistent homology sidesteps this entirely by considering all scales simultaneously. The filtration \(\mathrm{VR}(X, r_0) \subseteq \mathrm{VR}(X, r_1) \subseteq \cdots\) encodes the multi-scale structure. Features that persist across a wide range of scales are “real”; features that appear and die quickly are “noise.” This is the topological analogue of looking at a signal at multiple frequency bands rather than a single frequency.

In our engine: Each persistence pair \((b, d)\) becomes a delay tap in PersistenceDelay. Long-lived features (large \(d - b\)) echo for longer — significant topological features literally reverberate through time.

Application Trace: Persistence Diagrams and Stability

The stability theorem (Cohen-Steiner, Edelsbrunner, Harer 2007) states: \[ d_B(\mathrm{Dgm}(f), \mathrm{Dgm}(g)) \leq \|f - g\|_\infty \] where \(d_B\) is the bottleneck distance between persistence diagrams. This is a Lipschitz condition: small perturbations in the input produce small changes in the persistence diagram.

Why this matters for sonification: It guarantees that our audio output changes continuously when the point cloud is perturbed. If a user drags a point slightly, the sound doesn't jump discontinuously — it morphs smoothly. Without stability, the mapping from topology to sound would be brittle and unusable as a real-time instrument.

Broader applications: Stability is the reason TDA works on noisy real-world data. It has been applied to: shape recognition (robust to sampling noise), protein structure classification (invariant under thermal fluctuations), sensor network coverage verification (tolerant of position uncertainty), and brain connectivity analysis (stable across fMRI sessions).

Application Trace: Persistence Landscapes

The persistence diagram is a multiset, which is awkward for statistical operations (how do you compute a mean of multisets?). Bubenik (2015) introduced persistence landscapes \(\lambda_k(t)\): real-valued functions defined by sorting the tent functions of persistence intervals. Landscapes live in a Banach space, so you can add, average, and do functional analysis on them.

In our engine: PersistenceDiagram::landscape(dim, k, t) evaluates the \(k\)-th landscape function at parameter \(t\). This provides a continuous, differentiable signal that can directly modulate audio parameters — far smoother than using raw birth/death pairs.

Application Trace: Persistence Entropy

Persistence entropy \(H = -\sum p_i \log p_i\), where \(p_i = \mathrm{pers}_i / \sum \mathrm{pers}_j\), measures the “complexity” or “disorder” of the persistence diagram. High entropy: many features of similar persistence (complex, disordered topology). Low entropy: one dominant feature (simple, ordered topology).

In our engine: Entropy could control the density of the reverb, the number of active modulation sources, or the overall timbral brightness — a single scalar that captures “how topologically complicated is this space?”

Applications: Persistence entropy has been used in: distinguishing tumor types from histopathology images (Atienza et al. 2019), classifying time series (Perea & Harer 2015), and detecting phase transitions in materials (Hirata et al. 2020).

  2. Adams, H., Emerson, T., Kirby, M., et al. “Persistence images: a stable vector representation of persistent homology.” J. Mach. Learn. Res. 18(8) (2017), 1–35.
    An alternative vectorization of persistence diagrams using weighted Gaussian kernels. Could provide smoother audio parameter modulation than raw persistence pairs.
  3. Atienza, N., Gonzalez-Diaz, R., and Soriano-Trigueros, M. “On the stability of persistent entropy and new summary functions for topological data analysis.” Pattern Recognition 107 (2020), 107509. arXiv:1803.08304.
    Proves stability of persistence entropy and introduces normalized variants. Our persistence_entropy() follows the unnormalized definition from this paper.

Textbooks

  1. Edelsbrunner, H. and Harer, J.L. Computational Topology: An Introduction. AMS, 2010. Chapters VI–VIII.
    The standard textbook treatment of persistent homology, including the standard algorithm and the stability theorem.
  2. Oudot, S.Y. Persistence Theory: From Quiver Representations to Data Analysis. AMS Mathematical Surveys and Monographs 209, 2015.
    The algebraic foundations of persistence. The structure theorem for persistence modules (decomposition into interval modules) is the reason persistence diagrams are well-defined.
  3. Polterovich, L., Rosen, D., Samvelyan, K., and Zhang, J. Topological Persistence in Geometry and Analysis. AMS University Lecture Series 74, 2020.
    Connects persistent homology to symplectic geometry, dynamical systems, and function spaces. For the mathematically adventurous reader.
