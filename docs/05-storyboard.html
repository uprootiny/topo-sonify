Data Flow

The architecture uses three layers with clear ownership boundaries:

  1. Rust core (compiled to WASM): owns all mathematical computation and audio synthesis. Zero allocation in the audio path. All topology types are serde-serializable for cross-boundary communication.
  2. WASM bridge (via wasm-bindgen): thin wrapper that exposes the TopologySonifier API to JavaScript. Handles type conversion between Rust types and JsValue.
  3. Web Audio (JavaScript): an AudioWorkletProcessor runs on the audio rendering thread, calling into WASM for each 128-sample quantum. The main thread handles UI, rendering, and user interaction.

S.5 Interaction Storyboard

The following frames illustrate the user interaction flow, showing how topology changes drive sound transformations.

1

Empty canvas

Initial State

The canvas is empty. No audio is playing. The user is prompted to click to add their first point.

Topology: \(K = \emptyset\), all invariants undefined.

Sound: Silence.

2

First point

Single Point

The user clicks once. A single vertex appears.

Topology: \(\mathbf{f} = (1)\), \(\beta_0 = 1\), \(\chi = 1\).

Sound: A single sawtooth oscillator at the base frequency (e.g., 220 Hz). A pure, solitary drone. The reverb is short (positive curvature). No modulation, no sub-oscillators.

3

Two distant points

Two Components

A second point is placed far from the first (beyond the radius).

Topology: \(\mathbf{f} = (2)\), \(\beta_0 = 2\), \(\chi = 2\).

Sound: Two oscillator voices at slightly different frequencies. A two-note chord. The second voice enters with a brief attack envelope. The reverb becomes even shorter (\(\chi = 2\)). The listener hears two distinct tones—two components, two voices.

4

Points connected

First Edge (Component Merge)

The user drags one point closer, or increases the radius, until the two points connect.

Topology: \(\mathbf{f} = (2, 1)\), \(\beta_0 = 1\), \(\chi = 1\).

Sound: The two voices merge into one. The chord collapses to unison. The harmonic ratio \(f_1/f_0 = 1/2\) tunes the second oscillator to a sub-octave. The reverb lengthens. The listener hears unification—two becoming one.

5

Triangle boundary

A Loop Forms!

Three points form a triangle boundary (edges only, no fill).

Topology: \(\mathbf{f} = (3, 3)\), \(\beta_0 = 1\), \(\beta_1 = 1\), \(\chi = 0\).

Sound: A dramatic moment. An LFO modulator activates at 0.1 Hz, adding a gentle vibrato to the drone. The reverb transitions to medium length (\(\chi = 0\), flat/toroidal). The harmonic ratio \(3/3 = 1\) is unison. The listener hears a new dimension of motion—the loop has introduced periodicity into the sound.

6

Filled triangle

Loop Filled (Loop Dies)

Increasing the radius causes the triangle to fill (a 2-simplex appears).

Topology: \(\mathbf{f} = (3, 3, 1)\), \(\beta_0 = 1\), \(\beta_1 = 0\), \(\chi = 1\).

Sound: The LFO deactivates. The vibrato stops. The drone returns to a static pitch. The reverb shortens again. Harmonics shift: ratio \(3/3 = 1\), then \(1/3\). The listener hears the loop "die"—the periodic modulation vanishes as the hole is filled in.

7

Hollow tetrahedron

A Void Appears

Four points form the boundary of a tetrahedron (four triangular faces, no solid interior).

Topology: \(\mathbf{f} = (4, 6, 4)\), \(\beta_0 = 1\), \(\beta_1 = 0\), \(\beta_2 = 1\), \(\chi = 2\).

Sound: A sub-oscillator activates at \(f_0/2\), adding warmth and body. The harmonic ratios are \(6/4 = 3/2\) (perfect fifth!) and \(4/6 = 2/3\). The reverb is very short (\(\chi = 2\), highly curved sphere). The listener hears a richer, deeper sound—the void has become a resonant cavity.

S.6 Interaction Scenarios

Scenario A: "Growing a Circle"

The user places 8 points in a circle pattern, one at a time.

  1. Points 1-3: Components merge one by one. Chord narrows to unison.
  2. Points 4-6: Edges form a path. Still no loop. Static drone.
  3. Point 7: The path closes into a cycle! \(\beta_1 = 1\). LFO activates. Vibrato begins.
  4. Point 8: Strengthens the cycle. Perhaps additional loops form briefly.
  5. Increasing radius: Triangles fill in the circle. The loop dies. LFO deactivates. If radius is large enough, a void may form briefly.

Scenario B: "Pulling Apart"

Starting from a dense cluster (all connected), the user drags points apart.

  1. Initial: Dense complex with many simplices. Rich harmonic content, sub-oscillators active.
  2. As points separate: Higher simplices disappear first (tetrahedra, then triangles). Voids may appear briefly. The sound becomes "emptier" as complexity decreases.
  3. Further: Edges break. New components form. The drone splits into multiple voices—a chord emerging from unison.
  4. Final: All points isolated. \(\beta_0 = n\) voices, each at a slightly different frequency. A shimmering, dispersed soundscape. Maximum polyphony.

Scenario C: "Radius Sweep"

Points are fixed; the user sweeps the radius slider from 0 to max.

  1. \(\varepsilon = 0\): All isolated. \(n\) voices. No modulation. No reverb.
  2. \(\varepsilon\) increases: Components merge. Voices converge. Modulation appears as loops form.
  3. Middle range: Maximum topological complexity. Loops and possibly voids. Rich sound with modulation, sub-oscillators, complex delay echoes.
  4. \(\varepsilon\) large: Everything connected and filled. \(\beta_0 = 1\), \(\beta_1 = 0\). Single static drone. Minimal sound.

This sweep is the audible equivalent of the persistence filtration—the user literally hears the persistent homology.

S.7 Future Directions

Higher-Dimensional Complexes

The current implementation supports up to 3-simplices (tetrahedra). Extending to higher dimensions would capture \(\beta_3, \beta_4, \ldots\), which could map to additional audio channels (e.g., 3D spatialization from \(\beta_3\)).

Sheaves and Local-to-Global

Sheaves on simplicial complexes encode local data that glues together globally. A cellular sheaf assigns a vector space \(\mathcal{F}(\sigma)\) to each simplex and a linear restriction map \(\mathcal{F}(\sigma) \to \mathcal{F}(\tau)\) for each face relation \(\tau \leq \sigma\). Sheaf cohomology detects obstructions to global consistency.

For sonification: different "local instruments" could be assigned to different regions of the complex, with sheaf cohomology determining whether they harmonize globally. Non-trivial cohomology = dissonance.

Spectral Sequences

The Mayer-Vietoris spectral sequence and the Leray-Serre spectral sequence provide refined algebraic invariants that converge to the homology. Each page of the spectral sequence is a differential graded module. These could drive a multi-layered synthesis architecture where the "pages" correspond to successive levels of harmonic refinement.

Mapper and Reeb Graphs

The Mapper algorithm (a discrete approximation to the Reeb graph) produces a graph summary of high-dimensional data. The graph could drive a network of delay lines (waveguide mesh synthesis), with the Mapper graph topology determining the resonant modes of the network.

Real-Time GPU Acceleration

For larger point clouds, the VR complex construction and boundary matrix reduction could be accelerated using WebGPU compute shaders, allowing real-time sonification of point clouds with hundreds of points.

Collaborative Topology

Multiple users could contribute points to a shared point cloud over WebSocket, each hearing the same topological sonification but from different "listening positions" in the complex (different vertices chosen as the observation point, affecting the local sound via star and link computations).
